# Code of Conduct

PAREL is maintained as a focused technical project. Contributors are expected to keep discussions constructive, specific, and respectful.

## Expected Behavior

- Treat other contributors with respect.
- Keep feedback actionable and grounded in the code, documentation, or user impact.
- Assume public issue and pull request threads are shared workspaces.
- Disclose security issues through the process in [SECURITY.md](SECURITY.md), not public issues.

## Unacceptable Behavior

- Harassment, threats, or discriminatory language.
- Personal attacks or repeated bad-faith disruption.
- Publishing private information without consent.
- Publicly posting exploit details or credentials.

## Enforcement

Maintainers may edit or delete inappropriate comments, close issues or pull requests, request behavior changes, or block participation when needed to protect the project.

Report conduct concerns to the maintainers through the repository's private contact paths. Security-sensitive reports should use [SECURITY.md](SECURITY.md).
